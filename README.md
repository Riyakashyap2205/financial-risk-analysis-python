# financial-risk-analysis-python
Financial risk and customer behavior analysis using Python (EDA, segmentation, anomaly detection, hypothesis testing)

📊 Financial Risk & Customer Behavior Analysis

This project analyzes transactional banking data to uncover customer behavior patterns, account performance, and financial risk indicators.

🔍 Objective
Understand customer transaction behavior
Identify high-value and low-value accounts
Detect financial risk and anomalies
Perform hypothesis testing on customer segments

🛠️ Tech Stack
Python 🐍
Pandas
NumPy
Matplotlib / Seaborn
SciPy

📈 Key Insights
📉 System shows net outflow (debits > credits)
💤 ~47% accounts are inactive/dormant
💰 Only ~25% accounts have positive net inflow
🧠 Low-activity users hold higher balances (savers)
⚠️ Few accounts show high-risk behavior

📊 Visualizations
Monthly Credits vs Debits
Net Flow Trend
Customer Segmentation
Risk Distribution

🧠 Hypothesis Testing
H0: High-activity accounts do NOT have higher balances
H1: High-activity accounts have higher balances

👉 Result: Failed to reject H0
👉 Insight: Low-activity users maintain higher balances

📌 Conclusion
This analysis highlights customer engagement gaps, identifies high-risk accounts, and provides actionable insights for improving financial performance and monitoring.
